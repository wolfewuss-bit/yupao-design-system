视觉 / 阴影

阴影 Shadow

阴影用于表达界面元素的层级关系，帮助用户理解空间结构。鱼泡直聘使用三级阴影系统。

阴影层级 Shadow Levels

Shadow SM 0 1px 4px rgba(0,0,0,0.08)
Shadow MD 0 4px 12px rgba(0,0,0,0.12)
Shadow LG 0 8px 24px rgba(0,0,0,0.16)

阴影 Token 规范

Token亮色值暗夜值使用场景
shadow/sm0 1px 4px rgba(0,0,0,0.08)0 1px 4px rgba(0,0,0,0.24)卡片、列表项、输入框
shadow/md0 4px 12px rgba(0,0,0,0.12)0 4px 12px rgba(0,0,0,0.32)浮层、下拉菜单、Tooltip
shadow/lg0 8px 24px rgba(0,0,0,0.16)0 8px 24px rgba(0,0,0,0.40)弹窗、底部面板、模态框

Z-Index 层级

配合阴影使用的 z-index 层级规范，确保元素叠放顺序正确。

层级z-index使用场景
基础内容0页面正常内容流
悬浮元素10Sticky 头部、悬浮按钮
下拉菜单100Dropdown、Popover
导航栏200顶部导航、底部 TabBar
蒙层500遮罩层 Overlay
弹窗1000Modal、Dialog、ActionSheet
Toast2000轻提示、Snackbar
Loading3000全局加载指示器

使用规则

  • 暗夜模式：阴影透明度增大（约 3 倍），因为深色背景上需要更强的阴影才能感知层级
  • 避免滥用：阴影仅用于需要表达层级的元素，不要给所有卡片都加阴影
  • 层级一致：同一层级的元素使用相同的阴影值
  • 动态阴影：元素被按下时可降低阴影层级（如 MD → SM），增强交互反馈
