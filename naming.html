命名策略

用词专业严谨

产品用词符合行业标准，便于用户理解，不生造、不随意挪用其他语境的近似用词。

正确示范
简历投递
错误示范
简历发射

直白描述为主

对于绝大部分的常规性的功能与概念，简单直白地描述其概念或作用即可，过多的概念包装会增加用户认知与理解成本。

正确示范
职位推荐
错误示范
智能匹配引擎

适时使用概念包装

对于某些功能与概念，出于品牌宣传、运营推广等目的，可适当进行概念包装。

正确示范
鱼泡直聘
错误示范
招聘平台

考虑产品的长远定位

产品功能会不断地迭代演进，命名时需考虑其长远定位，保证其沿用性和扩展性。

正确示范
求职服务
错误示范
找工作

避免歧义与错误联想

命名时避免歧义，引起错误或负面的联想、谐音误读、近似字形误读等。

正确示范
薪资待遇
错误示范
工资条件

Token 命名规范

设计 Token 采用统一的命名格式，确保跨团队一致性。

格式：{类别}/{子类别}/{属性} 分隔符：/ (斜杠) 大小写：全小写，多词用连字符 (kebab-case) 语言：英文命名，中文注释 示例： color/primary/default → 主色 color/text/primary → 主文字色 spacing/lg → 大间距 16px radius/md → 中圆角 8px font/size/base → 基础字号 14px shadow/sm → 小阴影

组件命名规范

格式：{组件名}/{变体}/{状态} 示例： button/primary/default button/primary/pressed button/primary/disabled input/default/focus card/job/default tag/success/default
