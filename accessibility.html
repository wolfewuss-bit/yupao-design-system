无障碍 Accessibility

鱼泡直聘致力于为所有用户提供平等的使用体验，包括视觉障碍、运动障碍、认知障碍等特殊群体。

对比度要求 Contrast Ratio

遵循 WCAG 2.1 AA 标准，确保文字在背景上具有足够的可读性。

元素类型最低对比度WCAG 级别说明
正文文字（<18px）≥ 4.5:1AA主文字 rgba(0,0,0,0.85) 在白色背景上约 12.6:1 达标
大文字（≥18px Bold 或 ≥24px）≥ 3:1AA标题文字通常满足此要求
UI 组件边框≥ 3:1AA输入框边框、按钮边框等
非文本内容（图标）≥ 3:1AA功能性图标需满足对比度要求
禁用态元素无要求—禁用态不要求满足对比度标准

鱼泡直聘文字色对比度验证

Token亮色值白色背景对比度是否达标
color/text/primaryrgba(0,0,0,0.85)~12.6:1AA 达标
color/text/secondaryrgba(0,0,0,0.65)~7.5:1AA 达标
color/text/tertiaryrgba(0,0,0,0.45)~4.1:1注意: 仅大文字达标
color/text/disabledrgba(0,0,0,0.25)~2.2:1— 禁用态豁免

触控目标 Touch Targets

规范最小尺寸推荐尺寸说明
Apple HIG44 × 44 pt44 × 44 ptiOS 平台标准
Material Design48 × 48 dp48 × 48 dpAndroid 平台标准
WCAG 2.5.544 × 44 CSS px—Web 无障碍标准
鱼泡直聘44 × 44 px48 × 48 px统一标准，兼顾双平台
  • 按钮最小高度 44px，宽度根据内容自适应但不小于 44px
  • 列表项最小高度 52px，确保手指可准确点击
  • 相邻可点击元素间距不小于 8px，避免误触
  • 图标按钮视觉尺寸可小于 44px，但点击热区需扩展到 44×44px

语义化标记 Semantic Markup

规则说明示例
图片替代文本所有功能性图片必须提供 alt 属性<img alt="用户头像">
装饰性图片纯装饰图片使用空 alt<img alt="" role="presentation">
按钮标签图标按钮必须提供 aria-label<button aria-label="返回">
表单标签输入框必须关联 label<label for="phone">手机号</label>
页面标题每个页面有唯一的 title<title>职位详情 - 鱼泡直聘</title>
标题层级标题标签按层级使用，不跳级h1 → h2 → h3，不直接 h1 → h3
焦点管理弹窗打开时焦点移入，关闭时焦点回到触发元素Dialog 组件内置焦点管理

颜色不是唯一手段

  • 错误状态：红色边框 + 错误图标 + 错误文字提示（三重信息）
  • 成功状态：绿色背景 + 勾选图标 + 成功文字
  • 链接：蓝色文字 + 下划线（或明确的按钮样式）
  • 图表数据：颜色 + 图案/纹理 + 数据标签
