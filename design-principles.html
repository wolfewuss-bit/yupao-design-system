设计原则

鱼泡直聘的设计原则是所有设计决策的基石，指导我们在产品设计中做出一致、高质量的选择。

四大设计原则

一致性 Consistency

相同的功能在不同页面、不同平台上保持一致的视觉表现和交互行为。统一的设计语言降低用户学习成本，建立可预期的使用体验。

层级清晰 Clear Hierarchy

通过字号、字重、颜色、间距等视觉手段建立清晰的信息层级。用户能快速识别主要信息和次要信息，高效完成核心任务。

可访问性 Accessibility

设计面向所有用户，包括视觉障碍、运动障碍等特殊群体。确保足够的对比度、合理的触控区域、语义化的标记结构。

双主题适配 Dual Theme

所有设计决策同时考虑亮色与暗夜两套主题。通过语义化 Token 实现主题切换，确保两套主题下的视觉质量和可读性。

一致性详解

  • 视觉一致：相同类型的元素使用相同的颜色、字号、圆角、间距
  • 交互一致：相同的操作产生相同的反馈（如按下态、加载态）
  • 文案一致：相同概念使用相同的措辞（参见命名策略）
  • 跨平台一致：iOS 和 Android 保持核心体验一致，允许平台特性差异

层级清晰详解

层级手段实现方式示例
字号层级大字号 = 高优先级标题 18px → 正文 14px → 辅助 12px
字重层级粗字重 = 高优先级标题 SemiBold → 正文 Regular
颜色层级高不透明度 = 高优先级主文字 85% → 次文字 65% → 辅助 45%
间距层级大间距 = 区块分隔区块间 24px → 元素间 12px → 内部 8px
阴影层级强阴影 = 高层级弹窗 shadow/lg → 卡片 shadow/sm

可访问性详解

  • 对比度：正文文字对比度 ≥ 4.5:1（WCAG AA），大文字 ≥ 3:1
  • 触控区域：最小 44×44px，确保手指可准确点击
  • 颜色不是唯一手段：不仅用颜色传达信息，辅以图标、文字等
  • 动效可关闭：尊重系统"减少动态效果"设置
  • 语义化标记：使用正确的 HTML 语义标签，支持屏幕阅读器

双主题适配详解

  • 语义化 Token：使用 color/text/primary 而非 #000000
  • 色彩调整：暗夜模式下饱和度微调，避免刺眼
  • 阴影增强：暗夜模式阴影透明度增大约 3 倍
  • 图片适配：暗夜模式下图片可降低亮度 10–15%
  • 测试验证：每个设计稿同时输出亮色和暗夜两个版本
